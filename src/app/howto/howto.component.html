How to deposit NEP-5 GAS to NeoFS

Prepare your wallet

RPC endpoint http://85.143.219.93:30333 can be used for custom wallets.

All the examples below will be for neo-python.

Use pre-configured neo-python:

git clone https://github.com/nspcc-dev/neo-python

or get default neo-python: and change config file neo/data/protocol.privnet.json to this:

git clone https://github.com/CityOfZion/neo-python.git

Get Neo address

Run neo-python wallet

cd neo-python
python3.7 -m venv venv
source venv/bin/activate

Run pip install for first execution

pip3.7 install wheel -e .

rm -rf ~/.neopython/Chains/privnet*
np-prompt -p -v

Create wallet

neo> wallet create my_wallet.wallet

Now you can find NEO address and public key in the NeoFS TestNet:

neo> wallet verbose

Add NEP-5 GAS token information to the wallet

neo> wallet import token 9bcd6be0f0d2b731977640d14d7edf7a6b59ea77

Use `wallet export wif <address>` to get WIF for Neo address. For example:

neo> wallet export wif AUJ3GSGiKShHFnkjBuVKfmAE4xdmE4qN6a

Deposit NEP-5 GAS to NeoFS

To deposit GAS to the NeoFS smart contract need to run contract 0x91ffdcb4e420c897bcaf8878092452a191993b89 method `Deposit` with options: public key, GAS amount.

For example:

neo> sc invoke 0x91ffdcb4e420c897bcaf8878092452a191993b89 Deposit [b'03a4ae16128e052f079de2474d8c74ee9db0cf4562f2085669eb04bfb4d2a257b7', 50]

Run NeoFS CLI

Get neofs-local repository

git clone https://github.com/nspcc-dev/neofs-local.git
cd neofs-local

Volume dir is './data'. Add files to have access from docker. In the docker this dir will be available in the root - `/data`.

Run NeoFS CLI

make cli

Declare NeoFS endpoint for CLI

Can be declared your own NeoFS node or default as in example below:

bash-5.0# neofs-cli set host 85.143.219.93:8080

Declare your private key for CLI by WIF

bash-5.0# neofs-cli set key L1b9rz58fZC6yXjg7nGAeAGM5dt8hhpScXA9stWLPVtLMWUPMc7X

NeoFS CLI first steps

Check balance

bash-5.0# neofs-cli accounting balance

If you have not sent NEP-5 tokens by `Deposit` invocation of 0x91ffdcb4e420c897bcaf8878092452a191993b89 smart contract - do this.

Create container

bash-5.0# neofs-cli container put --rule 'SELECT 3 Node'

Upload object to container

bash-5.0# neofs-cli object put --cid <container ID> --file </path/to/file>

Create Storage group

bash-5.0# neofs-cli sg put --cid <container ID> --oid <object ID>

Any number of objec IDs can be added to the storage group creation command.

Get object

bash-5.0# neofs-cli object get --cid <container ID> --file </path/to/out/file>

Open object by HTTP gate url

https://send.fs.neo.org/<containe_ID>/<object ID>

Additional information is available in github HowTo file.
