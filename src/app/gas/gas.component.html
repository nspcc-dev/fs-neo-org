Incentive model

For incentive model and pricing we implement a free market. Each node declares how much it wants to receive as a reward for data storage services for the upcoming epoch. Placement function considers the declared price and prefers nodes with better prices. However, because nodes in NetworkMap declare key-value attribute describing their parameters, like geographical location, type of storage, capacity and other things also used in StoragePolicy, it leaves room for nodes that want higher price better or unique services to still get into the placement function result.

In short, if you are too greedy, you will not get users and will not get rewards. To have higher prices you need to satisfy market demand or provide something unique.

As you see in the example graph of price to capacity change, node needs to keep price "in market" to get most profit. this should guarantee the competitive price for data storage in NeoFS.

Neo GAS token input

NeoFS uses Neo GAS for all payments. To be able to pay for storage services in NeoFS, the one has to deposit some GAS to NeoFS SmartContract address. This event will be monitored by InnerRing nodes and reflected in user's account balance internally in NeoFS.

NeoFS InnerRing nodes keep track of all account using NeoFS and using dBFT2.0 consensus algorithm, the same as in Neo blockchain, agree on users' mutual settlements, reward distribution between node owners and other payment-related procedures.

For example some person may own few storage nodes and provide storage services, but at the same time use NeoFS for storing his backups in different geographical location. In this case rewards from his storage nodes my be used to pay for his backups automatically, without transactions on main NeoBlockchain.

Neo GAS token withdraw

But it's also possible to withdraw the GAS accumulated inside NeoFS to Neo Blokchain address. To do it, user has to issue withdraw call to NeoFS. InnerRing nodes will lock the desired amount and issue a cheque, signed by all InnerRing nodes. Then, user has to invoke NeoFS Smart contract, providing it with this cheque. If it will be verified successfully, SmartContract will send GAS to user's address and InnerRing nodes will reflect those changes in internal account balance. In cheque is not used during validity period, gas, locked for withdrawal, will be unlocked. Just like in real world.
